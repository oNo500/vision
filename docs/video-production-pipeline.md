# 短视频制作链路：从趋势监控到发布分发

对新手来说，这条链路比直播更容易获得正向反馈：
- 不需要出镜、不需要实时互动
- 热门趋势已验证市场，降低选题风险
- 批量制作快速积累发布量，提高撞上流量的概率

---

## 整体链路

```
趋势监控 → 选题决策 → 素材获取 → 视频制作 → 去重处理 → 上传分发
```

```
   自己账号能否快速复刻该格式？
   → 立即生产 / 预埋 / 放弃
```

### FFmpeg 批量处理

```bash
# 批量转换为竖屏 9:16
for f in *.mp4; do
  ffmpeg -i "$f" \
    -vf "scale=1080:1920:force_original_aspect_ratio=decrease,pad=1080:1920:(ow-iw)/2:(oh-ih)/2" \
    -c:a copy "output_${f}"
done

# 批量添加水印（右下角）
ffmpeg -i input.mp4 -i logo.png \
  -filter_complex "overlay=W-w-10:H-h-10" output.mp4

# 批量提取封面帧
for f in *.mp4; do
  ffmpeg -i "$f" -ss 00:00:01 -vframes 1 "${f%.mp4}_thumb.jpg"
done
```

## 四、内容复刻的合规边界

### 平台检测机制（2025-2026）

抖音采用三重 AI 检测：

| 检测维度 | 阈值 |
|---------|------|
| 画面指纹比对 | 相似度 > 70% 触发预警 |
| 音频声纹识别 | 0.1 秒内识别，3 秒片段即可触发 |
| 创意重合度分析 | 非原创素材 > 70% 且无叙事逻辑判违规 |

**已被 100% 识别、完全失效的手法：**
- 调整画面比例/加黑边
- 替换 MD5（画面指纹不变）
- 镜像翻转、变速、换音
- 片头片尾加黑屏、三明治帧包裹

### 合规边界

| 形式 | 合规条件 | 风险 |
|------|---------|------|
| 解说/评论 | 引用 ≤ 10 秒 + 原创解说 > 40% + 标注来源 | 低 |
| 混剪 | ≥ 3 个不同来源素材 + 原创叙事 | 中 |
| 重拍/复刻 | 完全原创素材，仅借鉴创意框架 | 低，最推荐 |
| 配音/翻译 | 需获得原视频授权 | 中高 |

> [!IMPORTANT]
> 直接搬运外网视频（哪怕加了字幕或配音）不属于二创，侵权风险不因"技术处理"而消失。

### 最安全的复刻方式

**模仿创意框架，不复制具体内容：**

1. 分析爆款的成功要素（主题、叙事结构、节奏节点）
2. 使用完全自主拍摄的素材，保留带时间戳的原片
3. 重构叙事逻辑，加入个人体验和视角
4. 文案重新撰写，引用信息注明来源

### 适合复刻的内容类型（风险低）

| 类型 | 原因 |
|------|------|
| 知识科普 | 知识本身不受版权保护，素材可自主制作 |
| 生活技巧 | 操作步骤可自拍复现 |
| 本地探店 | 场景唯一，内容天然原创 |
| 产品测评 | 使用自购产品自拍，素材完全自主 |
| 教程类（美食/手工） | 重拍全程，加入个人改良即可 |

> [!CAUTION]
> 高风险类型：影视解说（需授权）、翻唱（需双重授权）、体育赛事混剪、新闻时事搬运。

## 五、自动化上传与多平台分发

### 开源工具

**[social-auto-upload](https://github.com/dreammis/social-auto-upload)** — 9.7k Stars，首选
- 支持：抖音、小红书、视频号、TikTok、YouTube、Bilibili
- 基于 Playwright 控制浏览器，支持定时发布、Cookie 管理、标题/标签自动填写
- 2026-03 仍在活跃维护

| 项目 | Stars | 特点 |

| 工具 | 平台数 | 适合场景 |
|------|--------|---------|
| 聚媒通 | 60+ | 多账号矩阵，独家 IP 隔离防限流 |

### 推荐技术架构（自建）

```
视频文件目录
    ↓
配置文件（标题 / 标签 / 封面 / 发布时间）
    ↓
social-auto-upload（Python + Playwright）
    ├── 抖音 / 小红书 / 视频号
    └── TikTok / YouTube / Bilibili
    ↓
cron 定时任务
    ↓
Telegram Bot 通知发布结果
```

### 矩阵账号运营规范

> [!WARNING]
> 矩阵运营的核心是"隔离"而非"批量"，违规代价是累积性的。

**必须隔离的维度：**
- **设备**：一机一卡一号，每台手机独立 SIM 卡
- **网络**：不同账号不在同一 Wi-Fi 下切换登录
- **内容**：每个账号独立的定位、人设、发布节奏

**内容差异化三维度：**
- 人设层：专家 / 体验官 / 爆料者（完全不同角色）
- 角度层：同产品，A 号测评，B 号讲技巧，C 号聊行业
- 形式层：A 号口播，B 号情景剧，C 号图文快闪

**发布节奏：**
- 建议每日 1-3 条，避免被判定为"工业化批量发布"
- 各账号发布时间错开，不在同一时间点精确发布

## 六、风险速查

```
行为                              风险等级
──────────────────────────────────────────
直接搬运（任何技术处理）           极高  → 技术处理不消除法律责任
外网视频加字幕搬运                 极高  → 不属于二创
使用未授权 BGM（哪怕 3 秒）        高   → 0.1 秒内被识别
AI 批量生成同质化内容              高   → 2025 新规明确打击对象
同设备操作多账号                   高   → 关联封号风险
矩阵号发布相同视频                 高   → 全账号同质化限流
解说/评论类（引用≤10秒+原创>40%）  低中  → 相对安全
完全自拍素材的同类型内容            低   → 最安全
抖音平台内置曲库音乐               低   → 已获版权可直接用
──────────────────────────────────────────
```
