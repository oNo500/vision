# 直播 AI 集成风险评估

## 可集成的 AI 能力

| 能力 | 场景 | 成熟度 |
|------|------|--------|
| TTS（文字转语音） | 弹幕朗读、自动回复播报 | 高，开源方案成熟 |
| LLM 实时互动 | 弹幕问答、话题引导 | 高 |
| ASR 实时字幕 | 主播说话转文字上屏 | 中，Whisper 延迟约 1-2s |
| 数字人驱动 | 实时换脸、口型驱动 | 中，需要 GPU |

### 技术栈组合思路

```
弹幕数据层        UniBarrage / 官方开放平台 API
      ↓
AI 处理层         LLM（内容生成）+ Whisper（ASR）+ TTS
      ↓
输出层            虚拟声卡注入直播音频 / OBS 字幕插件 / 数字人渲染
```

---

## 风险分层

### 弹幕抓取层（高风险）

整个链路的源头，也是风险最集中的地方：

- 非官方逆向方案违反平台服务条款，`signature` 签名算法随时更新导致方案失效
- **平台层面**：直播间被限流、账号封禁
- **法律层面**：未经授权抓取数据，涉及《数据安全法》和网络爬虫相关合规风险

### AI 内容输出层（中等风险）

TTS 播报、LLM 回复在抖音有明确规定：

- 禁止 AI 合成声音冒充真人（2025 年新规）
- 直播间播出 AI 生成内容须**显著标注 AI 生成**，否则属于虚假信息
- LLM 回复内容若触发违禁词，与人工话术同等对待，AI 实时检测不会区分来源

### 数字人层（政策最严）

- 2023 年起施行《互联网信息服务深度合成管理规定》
- AI 生成的人脸、声音必须有显著标识
- 数字人直播带货属于重点监管对象，商业产品（闪剪、创客兔等）均在合规层做了大量工作

---

## 风险速查

```
高风险   非官方弹幕抓取         随时失效 + 法律风险
中风险   AI 声音/数字人未标注   平台封禁 + 监管处罚
低风险   本地离线工具           不接入平台，无直接风险
```

---

## 相对安全的集成路径

> [!TIP]
> 优先选择官方 API 接入，离线工具次之，自动化程度越高风险越高。

**1. 官方 API + TTS（推荐）**
走 B站开放平台 WebSocket 或抖音开放平台合规接入弹幕数据，再接 TTS 播报，全链路合规。

**2. 离线工具**
不直连平台，针对录制好的内容做 ASR 字幕、音频后处理，无直接风险。

**3. 人在回路（Human-in-the-loop）**
AI 只做辅助建议，主播决定最终输出内容。自动化程度低，合规风险最小，也是最可持续的方向。

> [!CAUTION]
> 自动化程度越高、越绕过平台官方渠道，封控风险就越高。工具设计时需在「能力边界」和「合规边界」之间找平衡点。
